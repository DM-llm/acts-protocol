| 模块 | 说明 | 入口文档 |

## 📝 维护指南

### 添加新文档

1. **创建文档**到合适的位置：
   - 架构文档：`{模块}/{功能}/01-架构设计.md`
   - 工作日志：`{模块}/{功能}/work-logs/YYYY-MM-DD-{TYPE}-{DESCRIPTION}.md`
     - `impl` - 新功能开发
     - `fix` - Bug 修复
     - `pr` - PR 评审反馈
     - `refactor` - 代码重构
   - 经验教训：`lessons/YYYY-MM-DD-{DESCRIPTION}.md`

2. **更新此 INDEX.md**：
   - 添加到"按模块"或"按关键词"部分
   - 更新统计信息

3. **更新模块 INDEX.md**（如果存在）：
   - 添加到模块的文档列表

### 更新现有文档

1. **更新文档**本身
2. **更新 INDEX.md**（如果）：
   - 文档移动到不同位置
   - 文档重命名
   - 添加了新关键词

## 🎯 最佳实践

### 对于 AI

**场景 1：理解一个功能**
```
1. 读取 {模块}/INDEX.md
2. 找到该功能
3. 读取架构文档
4. 查看 work-logs 了解上下文
```

**场景 2：避免过去的错误**
```
1. 按关键词搜索 lessons/
2. 读取相关经验教训
3. 应用到当前任务
```

### 对于开发者

**开始新功能**：
1. 创建 `{模块}/{功能}/` 目录
2. 添加 `01-架构设计.md`
3. 创建 `work-logs/` 子目录
4. 更新 INDEX.md

**记录工作**：
1. 记录到 `work-logs/YYYY-MM-DD-{TYPE}-{DESCRIPTION}.md`
   - `impl` - 新功能开发
   - `fix` - Bug 修复（反面教材）
   - `pr` - PR 评审反馈（团队规范）
   - `refactor` - 代码重构
2. 成熟后提炼到架构文档
3. 发现错误时创建 lesson
