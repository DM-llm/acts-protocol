## 📝 开发规范

### 代码风格
- 遵循 ESLint / Prettier 配置
- 使用 TypeScript 严格模式
- 组件命名采用 PascalCase
- 函数命名采用 camelCase

### Git 规范
- 分支命名：`feat/`, `fix/`, `refactor/`, `docs/`
- 提交信息：`type(scope): description`
- PR 必须经过 Code Review

### 测试要求
- 单元测试覆盖率 > 80%
- 关键功能必须有集成测试
- 提交前运行测试
